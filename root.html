Study on Accounting of TCP Retransmissions in Cellular Data Traffic

Why Root a Device?

We require the client device to be rooted or jailbroken in order to run a packet capture program (e.g. tcpdump, pirni). The method for rooting/jailbreaking the device differs depending on the device manufacturer, model and the cellular ISP. You may find the rooting procedures for your device via searching the web. If you need help on rooting the device, please send an email to yhwan@kaist.ac.kr.

Ex. Galaxy S3 (GT-I9300) (Note: Does Not Work for Other Devices!)

Reference: http://wccftech.com/guide-root-ubell6-android-412-jelly-bean-galaxy-s3-i9300/

1. Download and extract root firmware and Odin program to Windows PC for installing firmware to the client device.

Root Firmware: Download (Extract: CF-Auto-Root-m0-m0xx-gti9300.tar.md5)
Odin 3.07: Download (Extract: Odin3 v3.07.exe)

2. Switch off the device.

3. Restart the device in "Download Mode" by holding down on (Volume Down Button + Home Button + Power Button). Then press Volume Down Button to continue.

4. Run Odin v3.07.exe in administrator rights.

5. Connect the device to the PC via USB cable. Check a notification appear in Odin in the bottom left saying "Added!".

6. Click on PDA Button, and select CF-Auto-Root-m0-m0xx-gti9300.tar.md5 file.

7. Make sure that "Auto Reboot" and "F. Reset Time" boxes are checked.

8. Click on START Button to begin download. The device will automatically reboot if download was successful.

9. When a green PASS notification appears, unplug the device from the PC.
